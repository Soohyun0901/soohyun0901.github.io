# 01. 소개

버트온(VirtOn)는 Debian Linux 기반으로 하며, 가상 머신과 컨테이너를 실행하기 위한 플랫폼입니다. 두 가지 가상화 기술, 커널 기반 가상 머신(KVM)과 컨테이너 기반 가상화(LXC)를 구현하여 유연한 기술을 극대화합니다.<br><br>

버트온의 주요 설계 목표 중 하나는 관리를 최대한 쉽게 만드는 것이었습니다. 사용자는 버트온을 단일 노드에서 사용하거나, 여러 노드로 클러스터 구성할 수 있습니다. 모든 관리 작업은 웹 기반 관리 인터페이스를 통해 수행할 수 있으며, 초보 사용자도 몇 분 내에 버트온을 설정하고 설치할 수 있습니다.<br>

## 01-1. 중앙 관리

많은 사용자들이 단일 노드로 시작하지만, 버트온은 대규모 클러스터 노드로 확장할 수 있습니다. 클러스터 구성요소는 완전히 통합되어 있으며 기본 설치와 함께 제공됩니다.

* 고유한 멀티 마스터 디자인
    * 통합된 웹 기반 관리 인터페이스는 모든 KVM, LXC 등의 GuestOS 뿐만 아니라 클러스터 전체 내용을 제공합니다. 따라서 GUI에서 VM과 컨테이너, 스토리지 또는 클러스터를 쉽게 관리할 수 있습니다. 별도의 복잡하고 비싼 관리 서버를 설치할 필요가 없습니다.
* 프록스목스 클러스터 파일시스템(pmxcfs)
    * 버트온은 고유한 프록스목스 클러스터 파일시스템(pmxcfs)을 사용합니다. 이는 구성 파일을 저장하기 위한 데이터베이스 기반의 파일시스템이며, 이를 통해 수천 개의 가상 머신 구성을 저장할 수 있습니다. 이러한 파일들은 corosync를 사용하여 모든 클러스터 노드에 실시간으로 복제됩니다. 파일시스템은 모든 데이터를 디스크의 영구 데이터베이스에 저장하지만, 데이터의 복사본은 최대 30MB의 RAM에 상주합니다. 이는 수천 개의 VM에도 충분한 용량입니다.<br>
    버트온은 이 고유한 클러스터 파일시스템을 사용하는 유일한 가상화 플랫폼입니다.
* 웹 기반 관리 인터페이스
    * 버트온은 사용하기 쉽습니다. 관리 작업은 포함된 웹 기반의 인터페이스를 통해 수행할 수 있습니다. 즉, 별도의 관리 도구나 대용량 데이터베이스가 있는 추가 관리 노드를 설치할 필요가 없습니다. 멀티 마스터 도구를 사용하면 클러스터의 모든 노드에서 전체 클러스터를 관리할 수 있습니다. JavaScript Framework(ExtJS)를 기반으로 한 중앙 웹 기반 관리는 GUI에서 모든 기능을 제어하고 각 개별 노드의 이력 및 시스템 로그를 확인할 수 있게 합니다. 이 중앙 웹 기반 관리에는  백업, 복구 작업, 라이브 마이그레이션 또는 HA 트리거 활동 등이 포함됩니다.
* 명령줄 인터페이스(CLI)
    * Unix shell이나 Windows powershell이 익숙한 일부 사용자를 위해, 버트온은 가상 환경의 모든 구성 요소를 관리하기 위한 명령줄 인터페이스를 제공합니다. 이 명령줄 인터페이스는 지능적인 탭 완성 기능과 UNIX 매뉴얼(man) 페이지 형식의 전체 문서를 제공합니다.
* REST API
    * 버트온은 RESTful API를 사용합니다. 우리는 JSON을 주요 데이터 형식으로 선택했으며, 전체 API는 JSON 스키마를 사용하여 공식적으로 정의되어 있습니다. 이를 통해 맞춤형 호스팅 환경과 같은 타사 관리 도구를 빠르고 쉽게 통합할 수 있습니다.

# 02 버트온 설치

버트온은 Debian Linux를 기반으로 합니다. 이러한 이유로 버트온이 제공하는 설치 디스크 이미지(ISO 파일)에는 완전한 Debian Linux 시스템과 필요한 모든 버트온 패키지가 포함되어 있습니다.<br>
설치 프로그램은 설정 과정을 안내하며, 로컬 디스크 파티션 설정, 기본 시스템 구성(예: 시간대, 언어, 네트워크) 및 필요한 모든 패키지 설치를 지원합니다. 이 과정은 몇 분 이상 소요되지 않습니다. 제공된 ISO를 사용한 설치는 신규 및 기존 사용자 모두에게 권장되는 방법입니다.<br>
버트온은 기존 Debian Linux 시스템 위에 설치할 수도 있습니다. 이 옵션은 버트온에 대한 상세한 지식이 필요하기 때문에 숙련된 사용자에게만 권장됩니다.

## 02-1 시스템 요구사항
버트온을 프로덕션 환경에서 운영할 때는 고품질 서버 하드웨어를 사용하는 것을 권장합니다. 호스트 장애의 영향을 더욱 줄이기 위해, 고가용성(HA) 가상 머신과 컨테이너를 갖춘 클러스터로 버트온을 운영할 수 있습니다.<br>

버트온은 로컬 스토리지(DAS), SAN, NAS, 그리고 Ceph RBD와 같은 분산 스토리지를 사용할 수 있습니다. 자세한 내용은 스토리지 챕터를 참조하세요.

### 02-1-1 테스트를 위한 최소 요구사항
이러한 최소 요구사항은 오직 테스트 목적으로만 사용되어야 하며, 운영 환경에서는 사용하지 않아야 합니다.
* CPU: 64비트(Intel 64 또는 AMD64)
* KVM 전체 가상화 지원을 위한 Intel VT/AMD-V 지원 CPU/마더보드
* RAM: 1GB RAM, 게스트에 필요한 추가 RAM
* 하드 드라이브
* 하나의 네트워크 카드(NIC)

### 02-1-2 권장 시스템 요구사항
* Intel VT/AMD-V CPU 플래그가 있는 Intel 64 또는 AMD64
* 메모리: OS와 버트온 서비스를 위한 최소 2GB, 게스트를 위한 지정된 메모리 추가. <br>(Ceph와 ZFS의 경우 추가 메모리가 필요하며, 사용된 스토리지 1TB당 약 1GB의 메모리가 필요합니다.)
* 빠르고 이중화된 스토리지(SSD로 최상의 결과를 얻을 수 있습니다.)
* OS 스토리지: 배터리 보호 쓰기 캐시("BBU")가 있는 하드웨어 RAID 또는 ZFS용 비RAID(ZIL용 선택적 SSD) 사용
* VM 스토리지:
  - 로컬 스토리지의 경우, 배터리 백업 쓰기 캐시(BBU)가 있는 하드웨어 RAID 또는 ZFS와 Ceph용 비RAID 사용. <br>(ZFS와 Ceph 모두 하드웨어 RAID 컨트롤러와 호환되지 않습니다.)
  - 공유 및 분산 스토리지 가능
  - 우수한 성능을 위해 전력 손실 보호(PLP)가 있는 SSD 권장.<br>(일반 소비자용 SSD 사용은 권장하지 않습니다.)
* 선호하는 스토리지 기술 및 클러스터 설정에 따라 추가 NIC가 있는 이중화(멀티) Gbit NIC
* PCI(e) 패스스루를 위해서는 CPU가 VT-d/AMD-d 플래그를 지원해야 함

### 02-1-3 간단한 성능 개요
설치된 버트온 시스템의 CPU와 하드 디스크 성능 개요를 얻으려면, 포함된 pveperf 도구를 실행하세요.

> 이것은 매우 빠르고 일반적인 벤치마크일 뿐이며, 특히 시스템의 I/O 성능에 대해서는 더 자세한 테스트가 권장됩니다.

### 02-1-4 웹 인터페이스 접근을 위한 지원 브라우저
웹 기반 사용자 인터페이스에 접근하기 위해 다음 브라우저 중 하나를 사용하는 것을 권장합니다:
* Firefox, 현재 연도의 릴리스 또는 최신 장기 지원 릴리스
* Chrome, 현재 연도의 릴리스
* Microsoft의 현재 지원되는 버전의 Edge
* Safari, 현재 연도의 릴리스

모바일 기기에서 접근할 때, 버트온은 가벼운 터치 기반 인터페이스를 보여줍니다.

## 02-2 설치 미디어 준비
다음 주소에서 설치 ISO 이미지를 다운로드하세요: https://www.proxmox.com/en/downloads/proxmox-virtual-environment/-iso <br>
버트온 설치 미디어는 하이브리드 ISO 이미지이며, 두 가지 방식으로 사용할 수 있습니다:
* CD나 DVD에 굽기 위한 ISO 이미지 파일
* USB 플래시 드라이브(USB 스틱)에 복사하기 위한 원시 섹터(IMG) 이미지 파일<br>

## 02-3 설치 진행

버트온 ISO 이미지에는 다음과 같은 요소가 포함되어 있습니다:<br>
* 완전한 운영 체제(Debian Linux, 64비트)
* 로컬 디스크를 ext4, XFS, BTRFS, 또는 ZFS로 파티셔닝하고 운영 체제를 설치하는 인스톨러
* KVM 및 LXC 지원이 포함된 리눅스 커널
* 가상 머신, 컨테이너, 호스트 시스템, 클러스터 및 필요한 모든 리소스를 관리하기 위한 완전한 도구 세트
* 웹 기반 관리 인터페이스

> 참고
> 설치 과정 중 선택한 드라이브의 모든 기존 데이터가 제거됩니다.

준비된 설치 미디어(예: USB 플래시 드라이브 또는 CD-ROM)를 삽입하고 부팅하세요.

> 참고
> 서버 펌웨어 설정에서 부팅 미디어(예: USB)가 활성화되어 있는지 확인하세요. 버트온 버전 8.1 이전의 설치 프로그램에서 부팅할 때는 보안 부팅을 비활성화해야 합니다.

### 02-3-3 하드디스크 선택
이후 버트온을 설치할 하드 디스크를 선택합니다.
> 주의
> 기본적으로 서버 전체가 사용되며 기존의 모든 데이터가 제거됩니다. 설치를 진행하기 전에 서버에 중요한 데이터가 없는지 확인하세요.

*Options* 버튼을 사용하면 기본값이 ext4인 파일 시스템을 선택할 수 있습니다. 설치 프로그램은 파일 시스템으로 ext4 또는 xfs를 선택하면 LVM을 사용하며, LVM 공간을 제한하는 추가 옵션을 제공합니다(아래 참조).<br>
버트온은 ZFS에도 설치할 수 있습니다. ZFS는 여러 소프트웨어 RAID 레벨을 제공하므로, 하드웨어 RAID 컨트롤러가 없는 시스템에 대한 옵션입니다. 대상 디스크는 옵션 대화 상자에서 선택해야 합니다. 더 많은 ZFS 관련 설정은 고급 옵션에서 변경할 수 있습니다.

> 경고
> 하드웨어 RAID 위에 ZFS를 사용하는 것은 지원되지 않으며 데이터 손실을 초래할 수 있습니다.

### 02-3-4 기본 구성 설정
다음 페이지에서는 위치, 시간대, 키보드 레이아웃과 같은 기본 구성 옵션을 설정합니다. 위치는 업데이트 속도를 높이기 위해 가까운 다운로드 서버를 선택하는 데 사용됩니다. 설치 프로그램은 일반적으로 이러한 설정을 자동으로 감지할 수 있으므로, 자동 감지에 실패하거나 국가에서 일반적으로 사용되지 않는 키보드 레이아웃을 사용하려는 드문 상황에서만 변경하면 됩니다.

### 02-3-5 비밀번호 및 이메일 지정
다음으로 슈퍼유저(root)의 비밀번호와 이메일 주소를 지정해야 합니다. 비밀번호는 최소 5자로 구성되어야 합니다. 몇 가지 지침은 다음과 같습니다:<br>
* 최소 12자 이상의 비밀번호를 사용하세요.
* 소문자와 대문자, 숫자, 기호를 포함하세요.
* 문자 반복, 키보드 패턴, 일반적인 사전 단어, 문자 또는 숫자 시퀀스, 사용자 이름 그리고 생체 정보(예: ID 번호, 조상의 이름 또는 날짜)를 피하세요.<br>

이메일 주소는 시스템 관리자에게 알림을 보내는 데 사용됩니다. 예를 들어:<br>

* 사용 가능한 패키지 업데이트에 대한 정보.
* 주기적인 cron 작업의 오류 메시지.

이러한 모든 내용은 지정한 이메일 주소로 전송됩니다.

### 02-3-6 네트워크 구성
마지막 단계는 네트워크 구성입니다. 현재 업링크 상태인 네트워크 인터페이스는 드롭다운 메뉴에서 이름 앞에 채워진 원이 표시됩니다. 설치 중에는 IPv4 또는 IPv6 주소 중 하나만 지정할 수 있습니다. 이중 스택 노드를 구성하려면 설치 후 추가 IP 주소를 구성하세요.

### 02-3-7 최종 확인
다음 단계에서는 이전에 선택한 옵션들의 요약을 보여줍니다. 모든 설정을 다시 확인하고 변경이 필요한 경우 이전 버튼을 사용하세요.<br>

설치를 클릭하면 설치 프로그램이 디스크 포맷을 시작하고 패키지를 대상 디스크에 복사합니다. 이 단계가 완료될 때까지 기다린 다음, 설치 미디어를 제거하고 시스템을 다시 시작하세요.<br>
패키지 복사는 주로 설치 미디어의 속도와 대상 디스크 성능에 따라 몇 분 정도 소요됩니다.패키지 복사 및 설정이 완료되면 서버를 재부팅할 수 있습니다. 기본적으로 몇 초 후에 자동으로 재부팅됩니다.




## 02-4 설치 후 관리 인터페이스 접속
정상적으로 설치하여 재부팅이 완료되었다면, 버트온 웹 인터페이스를 사용하여 추가 구성을 할 수 있습니다.<br>
> 기본적으로 버트온 웹 인터페이스 접속은 *포트 8006* 을 사용합니다.

1. 브라우저에 접속하여 버트온 IP주소 + 포트 8006을 입력합니다.
  * ex) https://[VirtOn_IP_Address]:8006
2. 슈퍼 유저(root)의 계정 정보를 입력하여 로그인합니다.(realm PAM) 
3. IP 구성 및 호스트 이름을 확인합니다.
5. 시간대를 확인합니다.
6. 방화벽 설정을 확인합니다.
